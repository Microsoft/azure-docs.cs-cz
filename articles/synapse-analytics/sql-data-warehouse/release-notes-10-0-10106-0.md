# <a name="dedicated-sql-pool-formerly-sql-dw-in-azure-synapse-analytics-release-notes"></a>Vyhrazený fond SQL (dřív SQL DW) ve službě Azure synapse Analytics – poznámky k verzi

Tento článek shrnuje nové funkce a vylepšení v posledních verzích [vyhrazeného fondu SQL (dřív SQL DW)](sql-data-warehouse-overview-what-is.md) ve službě Azure synapse Analytics. V tomto článku jsou uvedeny také významné aktualizace obsahu, které přímo nesouvisejí s vydáním, ale publikované ve stejném časovém rámci. Vylepšení dalších služeb Azure najdete v tématu [aktualizace služby](https://azure.microsoft.com/updates).

## <a name="check-your-dedicated-sql-pool-formerly-sql-dw-version"></a>Zkontroluje vyhrazený fond SQL (dříve SQL DW) verze.

Protože se nové funkce zavádějí do všech oblastí, podívejte se na verzi nasazenou do vaší instance a nejnovější poznámky k verzi pro dostupnost funkcí. Pokud chcete zjistit verzi, připojte se k vyhrazenému fondu SQL (dřív SQL DW) prostřednictvím SQL Server Management Studio (SSMS) a spusťte příkaz `SELECT @@VERSION;` a vraťte aktuální verzi. Pomocí této verze můžete potvrdit, která verze se použila pro vyhrazený fond SQL (dřív SQL DW). Datum ve výstupu identifikuje měsíc, ve kterém se vydaná verze nastavila pro vyhrazený fond SQL (dřív SQL DW). To platí jenom pro vylepšení na úrovni služby. 

V případě vylepšení nástrojů se ujistěte, že máte ve verzi poznámky k verzi nainstalovanou správnou verzi. 

> [!NOTE]
> Název produktu vrácený funkcí SELECT @ @VERSION se změní z Microsoft Azure SQL Data Warehouse na Microsoft Azure synapse Analytics. Než provedete změnu, pošleme vám Pokročilá oznámení. Tato změna je relevantní pro zákazníky, kteří analyzují název produktu z výsledku SELECT @ @VERSION v kódu aplikace. Aby se zabránilo změnám kódu aplikace v důsledku přeručního přizpůsobení produktu, použijte prosím tyto příkazy k dotazování SERVERPROPERTY pro název a verzi databázového produktu: Pokud chcete vrátit číslo verze XX. X. XXXXX. X (bez názvu produktu), použijte tento příkaz:
>
> ```sql
> SELECT SERVERPROPERTY('ProductVersion')
>
> --To return engine edition, use this command that returns 6 for Azure Synapse Analytics:
>
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Zákazníci teď můžou data načíst pomocí externích tabulek z ABFS do vyhrazeného fondu SQL (dřív SQL DW). Tato funkce umožňuje zákazníkům integraci se svými datovými laky v Data Lake Storage Gen2.|
|**Významné chyby**|CETAS na Parquet selhání v malých třídách prostředků v datových skladech DW2000 a další – Tato oprava správně identifikuje odkaz s hodnotou null v tabulce vytvořit externí tabulku jako na Parquet cestu kódu.<br/><br/>Hodnota sloupce identity může v některých operacích CTAS přijít – při CTASed do jiné tabulky se nesmí zachovat hodnota sloupce pro identifikaci. Oznámeno na [blogu](https://blog.westmonroepartners.com/azure-sql-dw-identity-column-bugs/).<br/><br/>Vnitřní chyba v některých případech při ukončení relace v době, kdy je stále spuštěný dotaz – Tato oprava vyvolá chybu InvalidOperationException, pokud je relace ukončena, když je dotaz stále spuštěný.<br/><br/>(Nasazeno v listopadu 2018) Při pokusu o načtení několika malých souborů z ADLS (Gen1) pomocí základu pro zákazníky došlo k dosažení optimálního výkonu. – Při ověřování tokenu zabezpečení AAD byl výkon systému kritický. Problémy s výkonem se zmírnit tím, že povolíte ukládání tokenů zabezpečení do mezipaměti. |
